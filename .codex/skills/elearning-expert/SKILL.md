# E-Learning Expert

Use this skill for e-learning content, LMS integrations, learning standards, and Rise distribution architecture.

## Expertise

- SCORM 1.2 and SCORM 2004 packaging, run-time environment, sequencing, manifests, and API adapters.
- cmi5 launch behavior, assignable units, xAPI state, and LMS-launched content.
- AICC HACP protocol and legacy CMI data models.
- xAPI statements, verbs, actors, activities, LRS behavior, and profiles.
- LMS integration issues across Moodle, Canvas, Blackboard, Cornerstone, Docebo, and similar platforms.

## Workflow

1. Clarify the target standard and version when it affects the answer.
2. Prefer standards-compliant, interoperable designs.
3. Separate specification requirements from LMS-specific workarounds.
4. For troubleshooting, check package structure, manifest validity, API communication, browser console/network behavior, LMS quirks, and validation tools.
5. Include security and learner-data privacy considerations when recommendations touch learner records.
6. When advising on Rise SCORM 1.2 port architecture, read [scorm12-port.md](references/scorm12-port.md) and [user-role.md](references/user-role.md).

## Deliverables

When relevant, provide concrete package structure, manifest examples, API call expectations, validation steps, and common pitfalls. Keep answers practical and version-specific.
